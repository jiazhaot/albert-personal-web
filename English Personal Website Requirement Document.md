# 英文个人网站需求文档

# 1. 文档概述

## 1.1 目的

本文档详细阐述英文个人网站的所有功能及界面需求，作为开发工作的指导依据，确保开发完成的网站符合预期的布局、功能及用户体验，重点聚焦使用Elfsight Instagram Feed插件展示Instagram内容及个人内容展示相关需求。

## 1.2 范围

本需求范围涵盖个人网站首页的整体布局、侧边栏功能、主内容区域、导航交互、各模块（个人介绍、简历、摄影集）及使用Elfsight Instagram Feed插件展示Instagram内容的核心功能；不涉及后续网站维护、域名注册、服务器部署等非开发相关内容。

## 1.3 目标用户

希望了解网站所有者个人信息、工作经历、摄影作品的访客，包括同行、合作伙伴、朋友及其他对网站所有者感兴趣的人群。

# 2. 整体布局需求

网站采用双栏布局（固定布局，布局结构进行响应式调整），左侧为侧边栏，右侧为主内容区域，交互逻辑为“点击侧边栏导航，主区域显示对应内容”。

# 3. 侧边栏需求（左栏）

侧边栏固定在页面左侧，宽度固定（根据设计确定，尽量保证所有内容完整显示、不换行），高度与页面高度一致，内容从上至下按以下顺序排列：

## 3.1 个人基础信息区域

- 个人照片：显示清晰的个人头像（圆形图片，尺寸根据设计确定，确保清晰无失真）；
- 姓名：以英文显示网站所有者全名，字体突出（略大于侧边栏其他文本）；
- 职业：以英文显示网站所有者职业（食品科学家，摄影师），字体尺寸小于姓名，颜色较浅（与姓名形成区分）；
- 个人简介（简短）：1-2句英文简洁介绍，聚焦个人定位或核心兴趣（“一名专注于日常生活与自然风光的热忱摄影师”），字体简洁易读。

## 3.2 导航栏

位于个人基础信息区域下方，导航项以垂直列表形式展示，样式统一，当前选中项需高亮显示（如改变颜色、或者背景色）。导航项包含以下3项（英文显示，顺序固定）：

- 关于我（About Me）：对应主区域显示详细个人介绍内容；
- 我的简历（My CV）：对应主区域显示简历内容；
- 摄影集（Photo Gallery）：对应主区域显示摄影作品展示内容（使用Elfsight Instagram Feed插件，同步展示个人Instagram账号内容）。

## 3.3 联系方式图标区域

位于侧边栏底部（固定在底部，不随页面滚动），水平展示3个联系方式图标（尺寸统一、间距一致），每个图标可点击并跳转至对应链接。图标及对应链接如下：

- 领英（LinkedIn）图标：点击跳转至网站所有者领英个人主页（新标签页打开）；
- 照片墙（Instagram）图标：点击跳转至网站所有者Instagram个人主页（新标签页打开）；
- 邮箱（Email）图标：点击触发系统邮件客户端，自动填充网站所有者邮箱地址（供访客直接发送邮件）。

说明：图标需采用英文风格，简洁大方，与网站整体调性一致，禁止出现中文元素。

# 4. 主内容区域需求（右栏）

主内容区域占据页面右侧剩余空间（不含侧边栏宽度），内容根据点击侧边栏的导航项动态切换。整体风格简洁干净，全程显示英文内容，无中文元素，布局与侧边栏风格统一（简洁、大方、聚焦内容展示）。

## 4.1 内容显示规则

1. 默认规则：网站首次打开时，主区域默认显示“关于我”对应的内容（详细个人介绍）；
2. 切换规则：点击侧边栏不同导航项时，主区域内容即时切换（不刷新页面，建议添加流畅过渡效果）；
3. 滚动规则：主区域内容超出页面高度时可滚动，侧边栏保持固定（不随主区域滚动）。

## 4.2 “关于我”内容（详细个人介绍）

以英文显示详细个人介绍，结构清晰、内容丰富（不少于3个段落），包括但不限于以下方面：

- 个人背景（如与职业相关的教育经历、工作经历）；
- 核心兴趣爱好（重点体现与职业或摄影的关联，如有）；
- 个人理念或追求（如工作理念、摄影理念）；
- 网站所有者希望展示的其他内容（如个人成就、有趣经历）。

## 4.3 “我的简历”内容

以英文显示网站所有者的简历，结构清晰、分类展示、便于浏览。推荐结构如下（可根据实际简历内容调整）：

- 个人信息（简洁，与侧边栏一致，无需重复照片）；
- 职业概述（1个段落，总结职业经历及核心优势）；
- 工作经历（按倒序排列，包含公司名称、职位、工作时间、主要职责及成就）；
- 教育背景（按倒序排列，包含学校名称、专业、学历、毕业时间）；
- 技能（如专业技能、语言技能、软件操作技能）；
- 奖项与证书（如有，按倒序排列）；
- 其他信息（可选，如志愿者经历、个人项目）。

说明：布局简洁，不同模块区分清晰（如使用标题、分隔线），无多余装饰。

## 4.4 “摄影集”内容（核心功能：使用Elfsight Instagram Feed插件）

本模块为网站核心，需嵌入Elfsight Instagram Feed插件，实现同步展示网站所有者Instagram账号帖子的功能，替代原有API集成方式，插件使用需符合其官方规范，展示效果及功能需求如下：

### 4.4.1 插件集成要求

- 账号绑定与同步：通过Elfsight插件官方配置流程，绑定网站所有者个人Instagram账号，实现帖子实时同步（插件默认同步周期即可，确保网站展示内容与Instagram账号保持一致）；
- 插件适配：插件整体风格需与网站统一（简洁、大方、贴合英文个人网站调性），调整插件内置设置，确保无多余装饰、不破坏网站整体布局；
- 故障处理：若插件加载失败、账号绑定失效或同步异常，在摄影集区域显示友好提示（英文，如“Failed to load Instagram posts, please try again later”），不显示错误代码或杂乱内容；同步异常时，可通过重新配置插件绑定账号恢复。

### 4.4.2 帖子展示要求（依托Elfsight插件实现）

- 布局：借助插件网格布局功能，实现帖子2-3列展示（可根据屏幕尺寸自适应调整），帖子间距均匀、图片尺寸统一（无失真），贴合网站整体响应式设计需求；
- 帖子内容展示：通过插件配置，确保每个帖子卡片显示以下信息（均从绑定的Instagram账号同步，以英文展示）：                  帖子图片/视频：Instagram帖子的主图或封面（若为视频，需通过插件设置显示播放按钮，点击可在网站内直接播放，无需跳转至Instagram）；
- 帖子文案：Instagram帖子的文本内容（可通过插件设置“默认折叠、点击展开”，避免长文案占用过多空间）；
- 地点：帖子关联的位置信息（若Instagram帖子设置了地点，通过插件同步显示英文地点名称；未设置则不显示该模块）；
- 时间：帖子发布时间（通过插件设置为英文格式，如“Jan 15, 2026”，与Instagram时间显示风格一致）；
- 点赞数：帖子获赞数量（显示为“X Likes”，如“128 Likes”，插件内置点赞图标，保持与Instagram图标风格统一）。

交互设置：通过插件开启“帖子放大查看”功能，点击帖子卡片可全屏或弹窗放大图片/视频，放大后下方同步显示该帖子的完整信息（文案、地点、时间、点赞数），点击关闭按钮可返回网格布局；插件内置交互效果需流畅，无卡顿。

# 5. 风格要求

- 整体调性：简洁、大方、专业，贴合英文个人网站风格，避免花哨色彩及多余装饰；
- 色彩搭配：配色统一（不超过3种主色），推荐使用低饱和度色彩（如白色、浅灰色、深灰色、浅蓝色），侧边栏与主区域色彩区分清晰（如侧边栏浅灰色、主区域白色），文本颜色清晰易读；
- 字体：使用统一英文字体（如Arial、Helvetica），字体尺寸层次分明（标题>正文>辅助文本），不使用花哨字体；
- 响应式设计：适配不同屏幕尺寸（电脑、平板），侧边栏与主区域宽度自动调整，布局结构保持左右分布不变；主区域内容及Elfsight插件布局自动重排，确保正常显示。

# 6. 补充要求

- 网站所有内容均以英文显示，禁止出现中文字符及中文相关元素（包括图标、文本、链接等）；
- 网站加载速度较快（首页加载时间不超过3秒），重点优化Elfsight插件加载速度，可开启插件懒加载功能；
- 导航交互流畅，点击侧边栏导航切换主区域内容时，无卡顿、无延迟；
- Elfsight Instagram Feed插件使用需符合其官方服务条款及配置规范，避免因违规使用导致插件失效；
- 侧边栏联系方式图标显示正常，跳转链接有效（无无效链接）；
- 插件配置完成后，需测试不同设备、不同网络环境下的展示效果，确保同步稳定、交互流畅。

# 7. 验收标准

- 布局符合要求：左侧侧边栏、右侧主区域正常显示，侧边栏固定不动；
- 侧边栏所有模块完整显示，导航及联系方式图标可正常点击、功能有效；
- 主区域内容可根据侧边栏导航正常切换，各模块（关于我、我的简历、摄影集）内容符合需求；
- 摄影集模块成功嵌入Elfsight Instagram Feed插件，可正常绑定Instagram账号、同步帖子，帖子信息（图片/视频、文案、地点、时间、点赞数）完整显示；
- 网站风格统一，响应式设计正常，加载速度、交互流畅度符合要求；
- 所有链接有效，无交互错误，插件无加载异常、同步异常等问题。
